# Requirement 5: Demo Script for Nia

*Nia stands with the board while Amina prepares the release.*

Nia: Good morning. I’m going to show a safe release in real time, then how the system protects customers if something goes wrong.

*Amina deploys the new release to the green environment while the board watches the progress screen.*

Nia: You can see the new release being prepared in the background, with no interruption to customers yet.

*Amina switches live traffic to the green environment.*

Nia: Traffic is now moving to the new release, and the system is checking that everything is still healthy.

*Amina introduces the fault during the live run.*

Nia: Something has failed in the new release, and the system is now detecting that the service is not behaving as expected.

*The automated rollback fires and returns traffic to the previous environment.*

Nia: The system has recovered automatically and sent traffic back to the safe version in 13 seconds.

*Amina confirms the final state on screen.*

Nia: That matters because customers see a fast recovery, our team avoids manual intervention, and the business keeps trust when a release does not go as planned.
